## 📊 Dataset Overview

The **Olist Brazilian E-Commerce** dataset is one of the most comprehensive real-world e-commerce datasets publicly available. It encompasses **9 interrelated CSV files** covering the complete marketplace value chain — from customer registration through order delivery and post-purchase review.

| # | Table | Records | Cols | Description | Key Join Fields |
|:-:|:------|--------:|:----:|:------------|:----------------|
| 1 | **Customers** | 99,441 | 5 | Customer master data & demographics | `customer_id`, `customer_unique_id` |
| 2 | **Orders** | 99,441 | 8 | Central fact table with timestamps | `order_id`, `customer_id` |
| 3 | **Order Items** | 112,650 | 7 | Line-item details & pricing | `order_id`, `product_id`, `seller_id` |
| 4 | **Payments** | 103,886 | 5 | Payment method & installments | `order_id` |
| 5 | **Reviews** | 99,224 | 7 | Customer feedback & comments | `order_id` |
| 6 | **Products** | 32,951 | 9 | Product catalog & dimensions | `product_id` |
| 7 | **Sellers** | 3,095 | 4 | Seller directory & location | `seller_id` |
| 8 | **Geolocation** | 1,000,163 | 5 | Zip code lat/lng coordinates | `geolocation_zip_code_prefix` |
| 9 | **Category Translation** | 71 | 2 | Portuguese → English mapping | `product_category_name` |

### Analytical Tables Built

The pipeline constructs **two denormalized analytical tables** from the 9 raw sources:

| Output Table | Dimensions | Grain | Primary Use Cases |
|:-------------|:-----------|:------|:------------------|
| **Order-level Master** | 99,441 × 30 | One row per `order_id` | Revenue KPIs, churn modeling, delivery SLA, payment analysis |
| **Item-level Analytical** | 112,650 × 24 | One row per line item | Category drill-down, seller performance, market basket analysis |

> ⚠️ **Data Governance Rule**: A single order with multiple items must not double-count revenue. Order Items are aggregated to order-level *before* merging with the payment table. Reviews are deduplicated to keep only the latest per order.

### Core Metric Governance

| Metric | Formula | Business Definition |
|:-------|:--------|:--------------------|
| **Total Value** | `price + freight_value` | Sum per order, aggregated from line items |
| **Delivery Delay** | `actual_days − estimated_days` | Positive = late, negative = early delivery |
| **Late Delivery Flag** | `1 if delay > 0 else 0` | Binary indicator for SLA breach |
| **Churn Flag** | `1 if recency > 180 days else 0` | Customer inactivity threshold |
| **AOV** | `mean(total_order_value)` | Average order value across all orders |
| **CLV** | `RFM-based scoring` | Customer Lifetime Value for retention targeting |
| **Purchase Frequency** | `orders / customer` | Mean orders per unique customer |

## ❓ FAQ

<details>
<summary><strong>⏱️ How long does the full notebook take to run?</strong></summary>

The complete 36-section pipeline typically runs in **10–20 minutes** depending on hardware. The most compute-intensive sections are:
- **Section 31**: Advanced ML with Bayesian hyperparameter tuning (~3–5 min)
- **Section 33**: NLP Sentiment Analysis (~2–3 min)
- **Section 32**: Prophet + SARIMA forecasting (~2–3 min)
</details>

<details>
<summary><strong>🔀 Can I run individual sections independently?</strong></summary>

**Partially.** Sections 1–12 (Data Foundation & Feature Engineering) **must** be run sequentially first, as they build the master analytical tables. After that, most EDA and ML sections (13–36) can be run independently, though some have soft dependencies on prior feature engineering steps.
</details>

<details>
<summary><strong>📦 What if XGBoost/LightGBM/SHAP are not installed?</strong></summary>

The notebook includes **graceful fallbacks** using try/except blocks. If optional dependencies are not available, those sections will:
1. Print an informational message (e.g., `"XGBoost not available. Install with: pip install xgboost"`)
2. Skip the analysis
3. Continue to the next section without errors

The core pipeline (Sections 1–30) requires only `pandas`, `numpy`, `matplotlib`, `seaborn`, and `scikit-learn`.
</details>

<details>
<summary><strong>🐍 What Python version is required?</strong></summary>

**Python 3.10+** is recommended. The notebook uses f-strings, walrus operators, and other modern Python features. It has been tested on Python 3.10, 3.11, and 3.12.
</details>

<details>
<summary><strong>🌐 Do I need a Kaggle account to download the data?</strong></summary>

**Yes.** The Olist dataset is hosted on Kaggle at [olistbr/brazilian-ecommerce](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce). You need a free Kaggle account to download it. Alternatively, you can use the Kaggle CLI:
